# Genki Node Local MCP MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and locally validate an Agy-installable Genki Node MVP with one consent per bounded contribution session, hidden-by-default task details, automatic result delivery, and deterministic model-free cleanup.

**Architecture:** A single publishable TypeScript package contains focused `core`, `mcp-server`, and `cli` modules under `packages/`; the Agy plugin is a thin manifest/MCP/skill adapter. The CLI creates and authorizes a session, Agy drives the MCP tools over `stdio`, core persists only minimal crash-recovery state, and cleanup safely removes marked Genki-owned paths after delivery or session close.

**Tech Stack:** Node.js 22.18+, TypeScript 6.0.3, Zod 4.4.3, MCP TypeScript SDK 1.29.0, Vitest 4.1.10, ESLint 10.7.0, typescript-eslint 8.64.0, npm.

## Global Constraints

- The first host is Agy 1.1.2; Codex and Claude Code adapters are not part of this plan.
- MCP transport is local `stdio`; no TCP listener or daemon.
- One user authorization applies to one bounded contribution session; no per-task or result approval prompts.
- Task content is hidden by default in contributor-facing status, but is not claimed to be secret from the machine owner.
- Source repositories must be clean local Git repositories with no configured submodules or escaping symlinks.
- Validation commands use argument arrays and `shell: false`.
- Provider credentials and arbitrary parent environment variables never enter validation subprocesses or Genki persistence.
- Default contributor mode retains no per-task local history after cleanup.
- Cleanup is deterministic local code and must run with Agy unavailable and no model quota.
- Genki never edits undocumented Agy databases, history, `brain` directories, or provider/system logs.
- Federation One, Signal, GitHub automation, remote telemetry, Docker, and provider-key storage are excluded.

## File Map

- `package.json`: package metadata, binaries, dependency versions, and build/test/lint scripts.
- `tsconfig.json`, `tsconfig.build.json`, `eslint.config.js`: strict TypeScript build and lint configuration.
- `packages/core/src/types.ts`: public session, task, run, status, and result types.
- `packages/core/src/schema.ts`: Zod parsing for policies and local task files.
- `packages/core/src/digest.ts`: canonical JSON and SHA-256 digests.
- `packages/core/src/state-machine.ts`: valid session and task transitions.
- `packages/core/src/storage.ts`: atomic JSON state and ownership markers.
- `packages/core/src/cleanup.ts`: marker-checked task/session cleanup.
- `packages/core/src/repository.ts`: repository inspection, independent clone, and diff generation.
- `packages/core/src/environment.ts`: child-process environment allowlist and secret-name filtering.
- `packages/core/src/validation.ts`: direct validation process execution and bounded output.
- `packages/core/src/result.ts`: generic outcome, result bundle, and local test sink.
- `packages/core/src/engine.ts`: session/task orchestration used by CLI and MCP.
- `packages/core/src/index.ts`: stable core exports.
- `packages/mcp-server/src/server.ts`, `packages/mcp-server/src/bin.ts`: seven MCP tools and stdio entry point.
- `packages/cli/src/args.ts`, `packages/cli/src/consent.ts`, `packages/cli/src/agy.ts`, `packages/cli/src/bin.ts`: CLI parsing, one consent prompt, Agy launch, stop, status, and cleanup.
- `plugins/agy/plugin.json`, `plugins/agy/mcp_config.json`, `plugins/agy/skills/genki-contribution/SKILL.md`: installable Agy adapter.
- `tests/fixtures/repository/`: dependency-free local coding fixture generated by test helpers.
- `tests/local-harness/`: local task source/result sink used by integration and smoke tests.
- `README.md`: build, test, install, run, stop, privacy, cleanup, and known-limit instructions.

## Plan Self-Review

- Every approved design requirement maps to a task above.
- The session and task state names match the design specification.
- Cleanup precedes host packaging and is independently testable without Agy.
- Generic status types exclude task content; only the host-only prepared-task result contains instructions and workspace path.
- The real Agy test is last, after deterministic automated coverage, and does not replace it.
- No task introduces Federation One, Signal, remote APIs, Docker, GitHub automation, or other hosts.
